# Changelog

Notable changes per session. Newest on top.

## 2026-05-26 — net-wealth lean refactor + portfolio data hygiene + GitHub auto-push

### net-wealth dashboard

- **Tiles populating bug fixed.** `data/inputs.json` was using ad-hoc keys
  (`bank_qa_1_balance_qar`, `apartment_loan_inr`, `maa_tala_bari_inr`) that
  did not match the canonical keys generated by the form/view code
  (`bank_qa_0`, `loan_apartment`, `custom_maa_tala_bari`). Every override
  silently fell back to seed defaults.
- **Render abort bug fixed.** `render()` was calling
  `document.getElementById('cf-salary').textContent` on elements that had
  been removed from the HTML, raising a TypeError that aborted the rest
  of the render pipeline. KPI tiles, asset/liability reconciliation
  tiles, trend charts, todos, FIRE, and the edit form were all left
  blank. Setters now guard for element presence.
- **Lean refactor — single source of truth.** `data/inputs.json` deleted.
  `data/seed.json` is now the only data file. View, form, and save all
  read/write through `resolveKey()` which maps form keys directly to seed
  paths. localStorage caches the whole seed snapshot as `odin_state_v2`.
  Legacy `odin_manual_inputs_v1` auto-removed on load.
- **FX hero tile** pinned top-right of TOTAL NET WORTH. Shows live
  USD/INR and QAR/INR with source. QAR/INR derived from fixed 3.6413
  QAR/USD peg; USD/INR pulled from `portfolio/data/processed/market_indices.json`.
- **Load time 10-12s → instant.** Killed `loadGJCGoldRate` and
  `loadIBJARates` (each was doing 2 CORS-proxy fetches × 8s timeout
  inside `Promise.all`, gating render). Init now paints seed values
  immediately, then loadFX/loadPortfolio/loadHistoryFromRepo refresh in
  background and re-render on arrival. Firebase init non-blocking, 3000ms
  → 1200ms timeout.
- **Todo list resilient.** Falls back to `seed.todos` when localStorage
  holds an empty array, not just null.
- **Service worker** switched to network-first for shell, so future
  layout/script changes propagate without manual cache busting.
- **Cross-link to Portfolio** in bottom meta-bar (was always there).

### portfolio dashboard

- **Freshness badge fixed.** view() was picking the **MIN** per-ticker
  `as_of` for the global freshness timestamp. One delisted/failing
  ticker (ASHALOG, last good 2026-05-22) made the whole dashboard
  report `DATA 5266m OLD` while every other LTP was from today.
  Switched to MAX so the badge reflects the freshest update.
- **Stale-quote guard in `market_data.py`.** Yahoo's
  `regularMarketPrice` field gets stuck at an old value for some symbols
  (IRBINVIT.NS was frozen at ₹66.38 from 2024-07-23 while real current
  price was ₹60.52 — fake +9.76% day move). `fetch_quote_direct` now
  validates `regularMarketTime` against now() and falls back to the
  latest two historical closes when the realtime field is stale.
  Fallback chain reordered: direct chart API first (with freshness
  validation), yfinance history second, yfinance fast_info last (it's
  the bug surface).
- **Closed-position prune.** `market_data.py` now removes
  `holdings_prices.json` entries for tickers no longer in any open
  position. KEEL had been sitting there with a 3-day-old `as_of` after
  the position was closed.
- **India TODAY'S P&L** card now shows `≈ ±$X` USD equivalent under
  the ₹ value + % delta. Uses live FX with cost-basis `fx_inr_usd` as
  fallback.
- **Cross-link to Net Worth dashboard** under the logo in the top
  banner. Symmetric to the existing link in `net-wealth/`.

### GitHub Actions / workflows

- Deleted `ping_trigger.yml`. It relied on an external cron-job.org
  pinger that was never wired up. `full_update.yml` already has its own
  cron (every 15m during market hours) and `watchdog.yml` runs every
  5m, so it was redundant.
- Renamed `screener.yml` workflow title to force GitHub to re-index its
  `workflow_dispatch` trigger (the API reported it as missing despite
  YAML declaring it).
- Manually dispatched `eod_snapshot_us.yml` via API to clear its
  `no_success_ever` audit flag (was awaiting first scheduled run).

### Cloudflare Worker save proxy

- Added `tools/cloudflare-worker.js`. Worker accepts POST
  `{path, content}` from the dashboard origin, validates against a path
  whitelist, and commits via the GitHub Contents API using a PAT held
  server-side in the worker env.
- Deployed to `https://omm-money-saver.sabarna-chowdhury.workers.dev/`.
  Origin-locked to `https://tripurasundari-maa-sohay.github.io`.
  Whitelisted paths: `net-wealth/data/seed.json`,
  `portfolio/data/holdings_cost.json` (latter not yet wired).
- **net-wealth SAVE** now writes to localStorage AND fires a
  POST to the worker → seed.json auto-committed to repo on every save.
  No more manual download + commit step. Survives browser-storage
  wipes. PAT never lives in the browser.
- "Export to GitHub" button relabelled `☁ Push to GitHub` — same path,
  on-demand.
